# Modular Architecture Refactoring Plan - Synnel v2.0

**Date**: 2026-02-21
**Status**: Design Proposal
**Target**: Production-ready npm package release

---

## Executive Summary

This document outlines a comprehensive refactoring plan to transform Synnel into a modular, extensible, production-ready real-time synchronization library inspired by TanStack Query's architecture patterns.

**Vision**: A composable, type-safe real-time sync library where developers pick and choose the features they need, with clear separation of concerns and excellent developer experience.

---

## Current Architecture Analysis

### Current Structure

```
packages/core/src/
├── Client.ts                 # Browser client (258 lines)
├── Synnel.ts                 # Server class (126 lines)
├── Dispatcher.ts             # Event dispatcher (45 lines)
├── ConnectionManager.ts      # Reconnection logic (68 lines)
├── models/
│   ├── client.ts            # Server-side client model (67 lines)
│   ├── realm.ts             # Client registry (55 lines)
│   ├── channel.ts           # Channel model (34 lines)
│   ├── subscribe.ts         # Subscription manager (124 lines)
│   ├── message.ts           # Message types (52 lines)
│   └── transports.ts        # Transport abstractions (87 lines)
├── services/
│   └── socket.ts            # WebSocket server (134 lines)
└── types/
    └── index.ts             # Shared types (37 lines)
```

### Identified Problems

| Category | Issue | Impact |
|----------|-------|--------|
| **Modularity** | Client and Server tightly coupled through shared types | Hard to extend, difficult to tree-shake |
| **Separation** | No clear boundary between platform-specific code | Code meant for server imported in browser |
| **Abstraction** | Direct WebSocket dependency throughout | Cannot support other transports |
| **Testing** | No dependency injection, hard-to-mock dependencies | Zero test coverage |
| **Extensibility** | Fixed transport layer, no plugin system | Cannot add features without core changes |
| **Package Structure** | Everything in `@synnel/core` | Users cannot opt-in to features |

## Target Architecture (TanStack Query Inspired)

### Core Principles

1. **Core Minimalism**: Core package handles only the essential abstractions
2. **Platform Adapters**: Platform-specific code isolated in separate packages
3. **Plugin System**: Extensible through composable middleware/adapters
4. **Type-First**: Full TypeScript with exported utility types
5. **Testability**: Everything injectable, everything mockable

## Detailed Module Design

### 1. `@synnel/core` - Platform-Agnostic Core

**Purpose**: Define the abstractions and state management without transport details.

**Dependencies**: None

**Exports**:

```typescript
// core/channel.ts
export class Channel<T = any> {
  public readonly name: string
  protected state: ChannelState<T>

  subscribe(subscriber: string): void
  unsubscribe(subscriber: string): void
  broadcast(message: Message<T>): void
  getSubscribers(): Set<string>
  isEmpty(): boolean
}

// core/subscription.ts
export class Subscription {
  public readonly id: string
  public readonly channel: string
  protected state: SubscriptionState

  activate(): void
  deactivate(): void
  isActive(): boolean
}

// core/message-bus.ts
export class MessageBus {
  protected channels: Map<string, Channel>
  protected subscriptions: Map<string, Set<Subscription>>

  createChannel(name: string): Channel
  getChannel(name: string): Channel | undefined
  subscribe(channel: string, subscriber: string): Subscription
  unsubscribe(channel: string, subscriber: string): void
  publish(channel: string, message: Message): void
}

// core/protocol.ts
export enum MessageType {
  DATA = 'data',
  SIGNAL = 'signal',
  ERROR = 'error',
  ACK = 'ack',
}

export enum SignalType {
  SUBSCRIBE = 'subscribe',
  UNSUBSCRIBE = 'unsubscribe',
  PING = 'ping',
  PONG = 'pong',
}

export interface Message<T = unknown> {
  id: string
  type: MessageType
  channel?: string
  data?: T
  signal?: SignalType
  timestamp: number
}

// core/types.ts
export interface TransportConfig {
  url?: string
  reconnect?: boolean
  reconnectDelay?: number
  maxReconnectAttempts?: number
}

export interface TransportEvents {
  message: (message: Message) => void
  open: () => void
  close: () => void
  error: (error: Error) => void
}

export type TransportEventListener<K extends keyof TransportEvents> =
  (event: TransportEvents[K]) => void
```

### 2. `@synnel/client` - Browser Client (Framework-Agnostic)

**Purpose**: Browser-specific client implementation without framework coupling.

**Dependencies**: `@synnel/core`, `@synnel/adapter`

**Key Design**: Use factory functions and clean subscription APIs. No React dependencies.

```typescript
// client/index.ts
import { MessageBus } from '@synnel/core'
import { WebSocketTransport } from '@synnel/adapter'
import { ConnectionManager } from './connection-manager'

export interface CreateSynnelClientOptions {
  url?: string
  id?: string
  transport?: Transport
  reconnect?: boolean
  reconnectInterval?: number
  maxReconnectAttempts?: number
  onError?: (error: Error) => void
}

export function createSynnelClient(options: CreateSynnelClientOptions = {}) {
  const transport = options.transport ?? new WebSocketTransport({
    url: options.url ?? getDefaultUrl(),
    id: options.id,
  })

  const connection = new ConnectionManager(transport, {
    reconnect: options.reconnect ?? true,
    reconnectInterval: options.reconnectInterval ?? 1000,
    maxReconnectAttempts: options.maxReconnectAttempts ?? Infinity,
  })

  const bus = new MessageBus()

  return new SynnelClient({ bus, transport, connection })
}

export class SynnelClient {
  protected bus: MessageBus
  protected transport: Transport
  protected connection: ConnectionManager

  subscribe<T>(channel: string): ChannelSubscription<T>
  broadcast<T>(data: T): void
  onMessage(callback: (message: Message) => void): () => void
  onStatusChange(callback: (status: ConnectionStatus) => void): () => void
  disconnect(): void
  getStatus(): ConnectionStatus
}

// client/subscription.ts
export class ChannelSubscription<T = any> {
  protected listeners: Set<(data: T) => void> = new Set()
  protected errorListeners: Set<(error: Error) => void> = new Set()

  onMessage(callback: (data: T) => void): () => void
  onError(callback: (error: Error) => void): () => void
  send(data: T): void
  unsubscribe(): void
}

// client/types.ts
export type ConnectionStatus =
  | 'disconnected'
  | 'connecting'
  | 'connected'
  | 'disconnecting'

export interface ClientEvents {
  message: (message: Message) => void
  statusChange: (status: ConnectionStatus) => void
  error: (error: Error) => void
}
```

### 4. Transport Abstraction Layer

**Purpose**: Enable pluggable transports following the adapter pattern.

```typescript
// core/transport.ts
export interface Transport {
  readonly type: string
  readonly status: ConnectionStatus

  connect(): Promise<void>
  disconnect(): Promise<void>
  send(data: string): void

  on<K extends keyof TransportEvents>(
    event: K,
    listener: TransportEventListener<K>
  ): () => void
}

export type ConnectionStatus =
  | 'disconnected'
  | 'connecting'
  | 'connected'
  | 'disconnecting'

// adapters/websocket/client.ts
export class WebSocketTransport implements Transport {
  readonly type = 'websocket'
  protected socket?: WebSocket

  constructor(protected config: WebSocketConfig) {}

  async connect(): Promise<void> {
    return new Promise((resolve, reject) => {
      this.socket = new WebSocket(this.config.url)

      this.socket.onopen = () => {
        this.emit('open')
        resolve()
      }

      this.socket.onerror = (error) => {
        this.emit('error', error)
        reject(error)
      }

      this.socket.onmessage = (event) => {
        this.emit('message', event.data)
      }

      this.socket.onclose = () => {
        this.emit('close')
      }
    })
  }

  send(data: string): void {
    this.socket?.send(data)
  }

  async disconnect(): Promise<void> {
    this.socket?.close()
  }
}
```

## API Comparison

### Before (v1)

```typescript
// Server
import { Synnel } from '@synnel/core'

const synnel = new Synnel({ server, path: '/ws' })
const chat = synnel.multicast('chat')
chat.receive((data) => console.log(data))

// Client
import { Client } from '@synnel/core/client'

const client = new Client({ url: 'ws://localhost:3000' })
const sub = client.subscribe('chat')
sub.onMessage((data) => console.log(data))

// React
import { useChannel } from '@synnel/react'

function Chat() {
  const { data, send } = useChannel('chat')
  return <button onClick={() => send('hello')}>Send</button>
}
```

### After (v2)

```typescript
// Server - More explicit, better types
import { createSynnelServer } from '@synnel/server'

const server = createSynnelServer({
  server: httpServer,
  path: '/ws',
  channels: ['chat', 'notifications'],  // Whitelist channels
})

const chat = server.channel('chat')
chat.onMessage<ChatMessage>((data, client) => {
  console.log(`${client.id}: ${data.text}`)
})

// Client - Factory pattern, better DX, framework-agnostic
import { createSynnelClient } from '@synnel/client'

const client = createSynnelClient({
  url: 'ws://localhost:3000',
  reconnect: true,
})

const chat = client.subscribe('chat')
chat.onMessage((data: ChatMessage) => {
  console.log(data.text)
})

// React - Separate package, clean imports
import { useChannel, SynnelProvider } from '@synnel/react'

function App() {
  return (
    <SynnelProvider client={client}>
      <Chat />
    </SynnelProvider>
  )
}

function Chat() {
  const { data, send, status } = useChannel<ChatMessage>('chat', {
    onMessage: (data) => console.log('New:', data),
  })

  if (status === 'connecting') return <Spinner />

  return (
    <button onClick={() => send({ text: 'hello' })}>
      Send
    </button>
  )
}

// Vue - Future support, separate package
import { useChannel, SynnelProvider } from '@synnel/vue'

// Solid - Future support, separate package
import { useChannel, SynnelProvider } from '@synnel/solid'
```

## Key Improvements

### 1. Tree-Shakeable

```typescript
// Only import what you need
import { createSynnelClient } from '@synnel/client'
import { useChannel } from '@synnel/react'  // Separate React package

// vs before (everything bundled)
import { Client } from '@synnel/core/client'
import { useChannel } from '@synnel/react'
```

**Bundle Size Impact**:
- `@synnel/client` (framework-agnostic): ~10 kB
- `@synnel/react`: ~5 kB
- Total for React users: ~15 kB (vs ~25 kB before)
- Vue/Solid users only pay for what they use!

### 2. Transport Agnostic

```typescript
// Easy to swap transports
import { createSynnelClient } from '@synnel/client'
import { SSETransport } from '@synnel/adapter-sse'

const client = createSynnelClient({
  transport: new SSETransport({ url: '/events' })
})
```

### 3. Better TypeScript

```typescript
// Fully typed channels
interface ChatMessage {
  id: string
  text: string
  user: string
  timestamp: number
}

const chat = client.subscribe<ChatMessage>('chat')
chat.onMessage((msg) => {
  console.log(msg.user) // Fully typed!
})
```

### 4. Composable Middleware

```typescript
// Add logging middleware
import { logger } from '@synnel/devtools'

const server = createSynnelServer({
  server: httpServer,
  middleware: [logger()],
})

// Add auth middleware
const server = createSynnelServer({
  server: httpServer,
  middleware: [
    auth({ verify: async (token) => /* ... */ }),
  ],
})
```

## Breaking Changes

| Change | Migration Path |
|--------|---------------|
